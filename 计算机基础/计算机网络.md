### 0x00  计算机网络
在面试中，对计算机网络的考察很多，这里整理了100个高频问题。
### 0x01  什么是HTTP协议？
HTTP全称超文本传输协议，定义了客户端浏览器与服务端之间如何进行数据传输。它规范了客户端如何向服务器请求资源，以及服务器如何响应这些请求。包含多种请求方法，如常用的GET、POST等，每次请求和响应数据包可能包含了Status状态信息，Headers头部信息和可选的Body正文。传统的HTTP协议以明文形式传输，数据包容易被监听和篡改，所以现在一般使用更加安全的HTTPS。
### 0x02 什么是HTTPS？原理是什么？
HTTPS是HTTP的安全版本，通过TLS/SSL协议对数据加密。具体流程为：
1. 客户端向Web服务器发送HTTPS请求；
2. Web服务器返回公钥和CA数字证书；
3. 客户端验证证书的合法性和有效期；  

以上为证书验证过程，接下来为数据传输过程。

4. 客户端生成随机数，作为对称加密的密钥，利用公钥加密然后传给Web服务器；
5. 服务器通过私钥解密随机数，然后对传输数据进行加密；
6. 客户端接收加密数据，利用本地存储随机数解密，实现双方加密通信。
### 0x03 HTTP和HTTPS的区别
1. 端口号：HTTP默认80，HTTPS默认端口443；
2. URL前缀不同： http:// ,HTTPS是 https:// ；
3. 安全性：HTTP传输内容都是明文，客户端和服务端无法验证双方身份。HTTPS是运行在TLS/SSL上的HTTP协议，传输内容都经过了对称加密，密钥交换过程采用非对称加密，所以安全性更高；
4. 资源消耗：HTTPS高安全的同时会带来更多的资源消耗，加密解密过程会耗费更多的服务器资源；
5. SEO搜索引擎优化：搜索引擎通常会更青睐使用HTTPS协议的网站，因为HTTPS能够提供更高的安全性和用户隐私保护。
### 0x04 GET和POST的区别
1. 首先，GET请求没有body，参数包含在URL中，POST请求信息在Request body中；
2. 安全性上，POST比GET更安全，GET参数在URL中，会被记录到浏览器记录和日志中，而POST请求参数在body中，可以用不同方式加密。
3. 用途上，GET一般用来查询，POST用来给服务器发送数据，比如注册或文件上传等；
4. 接口一般用POST请求，而且是JSON格式。

### 0x05 从浏览器输入URL到页面展示过程中发生了什么？⭐
1. 首先在浏览器中输入指定网页的URL；
2. 浏览器通过DNS协议获取域名对应的IP地址（DNS查找过程）；
3. 根据IP地址和对应端口号，向目标服务器发起一个TCP连接请求；
4. 浏览器在TCP连接上，向服务器发起一个HTTP请求报文；
5. 服务器收到HTTP请求后，处理请求，并返回响应报文给浏览器；
6. 浏览器收到响应报文后，解析响应体中的HTML代码，渲染网页结构和样式，同时根据其他资源的URL再次发起HTTP请求，获取这些资源的内容，直到网页完全加载显示。

### 0x06 OSI七层模型
OSI七层模型自下而上依次是物理层、数据链路层、网络层、传输层、会话层、表示层、应用层。
1. 物理层：传输比特流；
2. 数据链路层：帧编码和误差纠正校验；
3. 网络层：路由和寻址；
4. 传输层：为两台主机进程间通信提供数据传输服务；
5. 会话层：管理会话；
6. 表示层：加密解密编码等数据处理；
7. 应用层：为用户提供各种服务。  

每一层专注自己的事情，并且利用下一层提供的服务。  
网络分层使各层之间相互独立，提高了灵活性和可替代性，把复杂问题分解成小问题来解决。
### 0x07 常见应用层协议
1. SMTP简单邮件发送协议：基于TCP，只负责邮件发送，不负责接收；
2. POP3/IMAP邮件接收协议：基于TCP，负责邮件接收。IMAP 支持邮件搜索、标记、分类、归档等高级功能，而且可以在多个设备之间同步邮件状态；
3. FTP文件传输协议：基于TCP：是用于在计算机之间传输文件的协议，但是明文传输，传输敏感数据建议使用SFTP；
4. Telnet远程登录协议：基于TCP，明文传输，所以一般使用SSH；
5. SSH安全网络传输协议：基于TCP，通过加密和认证机制实现安全的访问和文件传输等业务；
6. DNS域名管理系统：基于UDP，解决域名和IP地址映射问题。
### 0x08 常见传输层协议
- TCP传输控制协议，面向连接的，提供可靠数据传输服务；
- UDP用户数据协议，无连接的，提供尽最大努力的数据传输服务。
### 0x09 常见网络层协议
- IP：属于网络层的协议，主要作用是定义数据包的格式、对数据包进行路由和寻址，以便它们可以跨网络传播并到达正确的目的地，目前IP协议分为IPv4和IPv6；
- ARP地址解析协议：ARP协议解决的是网络层地址和链路层地址之间的转换问题。因为一个IP数据报在物理传输的过程中，需要知道下一跳该去哪，但IP地址属于逻辑地址，MAC地址才是物理地址，ARP协议解决了IP地址转换为MAC地址的问题；
- ICMP互联网控制报文协议：一种用于传输网络状态和错误消息的协议，常用于网络诊断和故障排除，Ping使用了ICMP协议来测试网络连通性；
- NAT网络地址转换协议：应用于内网与外网之间的网络地址转换过程中。
- OSPF开放式最短路径优先：内部网关协议，动态路由协议，基于链路状态算法，考虑了链路的带宽、延迟等因素来选择最佳路径；
- RIP路由信息协议：内部网关+动态路由协议，基于距离向量算法，使用固定的跳数作为度量标准，选择跳数最少的路径作为最佳路径；
- BGP边界网关协议：路由选择协议，在路由选择域之间交换网络层可达性信息。
### 0x10 简要介绍三次握手和四次挥手⭐
TCP采用三次握手策略建立连接。  

三次握手原理：  

第1次握手：客户端发送一个带有SYN标志的数据包给服务端；  

第2次握手：服务端接收成功后，回传一个带有SYN/ACK标志的数据包传递确认信息，表示收到；  

第3次握手：客户端再回传一个带有ACK标志的数据包，表示收到，握手结束。  

其中：SYN标志位置1，表示建立TCP连接；ACK标志表示验证字段。
  
  四次挥手原理：  

第1次挥手：客户端发送一个FIN，用来关闭客户端到服务端的数据传送，客户端进入FIN_WAIT_1状态；  

第2次挥手：服务端收到FIN后，发送一个ACK给客户端，确认序号为收到序号+1（与SYN相同，一个FIN占用一个序号），服务端进入CLOSE_WAIT状态；  

第3次挥手：服务端发送一个FIN，用来关闭服务端到客户端的数据传送，服务端进入LAST_ACK状态；  

第4次挥手：客户端收到FIN后，客户端t进入TIME_WAIT状态，接着发送一个ACK给Server，确认序号为收到序号+1，服务端进入CLOSED状态，完成四次挥手。  

其中：FIN标志位数置1，表示断开TCP连接。

### 0x11 TCP和UDP的区别⭐
1. 面向连接：UDP 在传送数据之前不需要先建立连接。而 TCP 提供面向连接的服务，在传送数据之前必须先建立连接，数据传送结束后要释放连接；
2. 可靠传输：远地主机在收到 UDP 报文后，不需要给出任何确认，数据是否丢失、是否顺序到达无法保证。TCP 提供可靠的传输服务，通过三次握手来建立连接，在数据传递时，有确认、重传和拥塞控制机制，保证数据正确有序到达；
3. 状态：TCP会记录自己消息发送的状态，但是UDP不会；
4. 效率：TCP由于确认和重传等各种机制，效率会比UDP低；
5. 传输形式：TCP面向字节流，UDP面向报文。
### 0x12 IPv4和IPv6的区别
IPv4是目前广泛使用的IP地址，使用32位地址，格式是4组由点分隔的数字组成。IPv6是为了解决IPv4地址耗尽的问题，使用128位地址，格式是由双冒号分隔的一组数字和字母。IPv6可以实现无状态地址自动配置，无需依赖DHCP，简化网络配置与管理。因为地址资源充足，无需再使用NAT了。
### 0x13 URL和URI的区别
URI：统一资源标志符，可以唯一标识一个资源；  
URL：统一资源定位符，可以提供该资源的路径，是一种具体的URI。即URL可以用来标识一个资源，还指明了如何locate这个资源。  
URI 的作用像身份证号一样，URL 的作用更像家庭住址一样。URL 是一种具体的 URI，它不仅唯一标识资源，而且还提供了定位该资源的信息。
### 0x14 Cookie、Session 和 Token
Session 的主要作用是通过服务端记录用户的状态。典型的应用场景是购物车，因为HTTP协议是无状态的，因此发送请求时服务器不知道是哪个用户在操作，因此服务端给特定用户创建特定session后，就可以标识用户并跟踪操作。  
Cookie数据保存在客户端（也就是浏览器端），用来保存用户的状态信息和偏好设置，还可以进行会话跟踪。网站通过cookie记录用户登录状态，避免每次访问时需要重新登录，还可以存储记录用户偏好等非敏感信息，每次访问时自动加载。进行 Session 认证的时候，一般使用 Cookie 来存储 SessionId。  
Token是一种加密字符串，用于客户端与服务器之间的用户认证。每次请求时，服务器通过验证 Token 来确认用户身份，从而实现无状态认证。Token适用于Web、移动端等不同平台，方便在分布式系统中使用。Token内容可以通过加密和签名保护。
### 0x15 简要概括Cookie、Session 和 Token 的区别
Cookie、Session 和 Token 通常都是用来保存用户登录信息的技术，但三者存在区别。简单来说 Cookie 适用于简单的状态管理，Session 适用于需要保护用户敏感信息的场景，而 Token 适用于状态无关的身份验证和授权。Cookie 存储在客户端，敏感信息存储需要进行加密处理。Session 存储在服务器端，通过一个 Session ID 在客户端和服务器之间进行关联。Token 通常使用加密算法生成，有效期较短且单向不可逆。
### 0x16 DNS的作用
DNS（Domain Name System）域名管理系统，是当用户使用浏览器访问网址之后，使用的第一个重要协议。DNS 要解决的是域名和 IP 地址的映射问题。在一台电脑上，可能存在浏览器 DNS 缓存，操作系统 DNS 缓存，路由器 DNS 缓存。如果以上缓存都查询不到，那么 DNS 就该发挥作用了，DNS 服务器把查询到对应域名的IP返回给用户。目前 DNS 的设计采用的是分布式、层次数据库结构，DNS 是应用层协议，它可以在 UDP 或 TCP 协议之上运行，端口为53。
### 0x17 DNS服务器有哪些？根服务器有多少个？
有三种类型的DNS服务器，分别是根DNS服务器、顶级域DNS服务器、权威DNS服务器。  
根DNS服务器：有400多个根DNS服务器遍布全世界，由13个不同组织管理，根服务器提供顶级域服务器的IP地址；  
顶级域DNS服务器：对于每个顶级域com、edu、gov等，和所有国家的顶级域uk、ca等都有顶级域名服务器或服务器集群，顶级域服务器提供了权威DNS服务器的IP地址；  
权威DNS服务器：在因特网上具有公共可访问主机的每个组织机构必须提供公共可访问的DNS记录，这些记录把主机名映射为IP地址。一个组织机构的权威DNS服务器收藏了这些DNS记录。  
本地DNS服务器：每个ISP都有一台本地DNS服务器，当主机发出DNS请求时，该请求先被发往本地DNS服务器，他起到一个代理的作用，并将该请求转发到DNS服务器层次结构中。
